Why Whispy

100% local

Audio is transcribed on-device with Whisper. Your voice never touches a server.

One key, anywhere

Hold your push-to-talk key to record, release to insert text into any focused field — terminal, browser, editor.

macOS & Linux

Native on macOS and Linux (X11). OS-specific seams sit behind a ports-and-adapters layer bound at runtime.

Multilingual

Defaults to French and handles many languages through the Whisper model you choose.

Lives in the tray

Runs quietly as a background daemon in the menu bar (macOS) or system tray (Linux). No window to manage.

Tunable models

Trade speed for accuracy by picking a Whisper model size that fits your machine.

How it works

  1. 1

    Hold your key

    A soft chime confirms Whispy is recording — Fn on macOS, Right Ctrl on Linux by default.

  2. 2

    Speak

    Whispy listens — the tray shows it's live (macOS adds an on-screen waveform).

  3. 3

    Release

    Whispy transcribes locally and types the text into the active field.

Install in one line

Runs on macOS and Linux (X11). Audio capture uses sounddevice/PortAudio — no sox needed.

curl -fsSL https://raw.githubusercontent.com/albrones/whispy/main/scripts/bootstrap.sh | bash

On macOS, prefer Homebrew?

brew install albrones/whispy/whispy
brew services start whispy

On Linux, install the system deps first (X11 session required — Wayland users pick an "Xorg"/"X11" session at login):

# Debian/Ubuntu — xdotool (text injection), xclip (optional clipboard), PortAudio
sudo apt install xdotool xclip libportaudio2
curl -fsSL https://raw.githubusercontent.com/albrones/whispy/main/scripts/bootstrap.sh | bash
